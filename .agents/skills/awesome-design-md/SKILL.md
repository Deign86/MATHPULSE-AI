Use this skill as the default visual direction source for frontend work.

Primary source: https://github.com/VoltAgent/awesome-design-md

## How to Apply

1. Pick a DESIGN.md profile from the awesome-design-md collection that matches the feature context.
2. Translate that profile into the project stack (React 18 + TypeScript + Tailwind CSS + Radix UI + Motion for React).
3. Keep visual decisions consistent with the chosen profile across typography, color, spacing, component styling, and motion.
4. Ensure responsive behavior and accessibility are preserved while applying the aesthetic direction.

## Implementation Expectations

- Use distinctive typography and avoid generic default font stacks.
- Use a clear color system with purposeful contrast and hierarchy.
- Use motion intentionally for key moments (entry transitions, staggered reveals, meaningful hover/focus states).
- Build depth with layered backgrounds, surfaces, borders, and shadow/elevation choices.
- Keep UI production-ready: typed, maintainable, and integrated with existing architecture patterns.

## Guardrails

- Do not copy proprietary logos, assets, or trademarked brand content.
- Use design language as inspiration and translate it into original implementation details.
- For mixed tasks, scope this skill to UI/frontend surfaces only.
